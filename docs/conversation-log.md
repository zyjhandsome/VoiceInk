## 第二轮：项目探索 & 第一个问题 - 语音识别引擎选择

**AI：**
项目目录目前是空的，这是一个全新的项目。已了解基本需求：
- 语音转文字的桌面软件
- 类似微信最新版的语音转文字功能
- 快捷键 Ctrl+Win 启动录音/转文字
- 带有 LLM 润色功能，用户可自选大模型
- 用户需配置 API URL、API Key、Model Name

**第一个问题：语音识别（Speech-to-Text）引擎选择？**

- **A. OpenAI Whisper API（在线）** — 识别精度高，支持多语言，需要网络，按时长计费。
- **B. 本地 Whisper 模型（离线）** — 使用 `faster-whisper` 或 `whisper.cpp` 在本地运行，无需网络，免费，但需要一定的 GPU/CPU 资源。
- **C. 系统原生语音识别（Windows Speech API）** — 零依赖，速度快，但中文识别精度一般。
- **D. 让用户自己选择** — 在设置中提供多种引擎选项。

初始推荐：A（OpenAI Whisper API），理由是实现简单、精度最高、与 LLM API 配置模式一致。

## 第三轮：用户追问 & 推荐更新

**用户：**
1. 有了语音识别引擎之后，是不是就不需要用户配置大模型信息了？
2. 软件可能要推广出去，如果对方电脑网络涉及信息安全限制，推荐用哪种方案？

**AI 解答：**

**关于语音识别引擎和大模型的关系：** 两者是不同步骤，都需要：
1. **语音识别引擎（STT）** — 把语音转成"原始文字"
2. **大模型（LLM）** — "润色"原始文字，使其更通顺

即使用了 Whisper API 做语音识别，用户仍需配置大模型信息来做润色。两者各司其职。

**关于推广和信息安全：** 更新推荐为 **B（本地 Whisper 模型）**，理由：
- 语音数据不出本机，满足信息安全诉求
- 无需额外 API 付费，降低推广门槛
- 离线可用，内网环境友好
- LLM 润色为可选功能，用户可配置内部允许的大模型 API

技术推荐使用 `faster-whisper`（Whisper 的高性能 C++ 实现的 Python 封装，CPU 上也能跑得很快）。

### 3. 代码审查 & 修复

派遣代码审查子代理，发现并修复了以下问题：

**修复的关键问题：**
1. ✅ SpeechRecognizer 并发模型访问 — 添加了运行状态检查，避免并发转写
2. ✅ FloatingWindow.show_error() 不显示窗口 — 添加了 position + show 调用
3. ✅ HotKeyManager.update_hotkey() 线程竞争 — 添加了锁保护
4. ✅ SettingsWindow 不重新加载设置 — 每次打开时重新加载配置
5. ✅ 空快捷键导致任意键触发录音 — 添加了非空检查
6. ✅ LLM 超时与设计文档不一致 — 修正为 5 秒
7. ✅ 测试连接阻塞主线程 — 改为异步线程执行
8. ✅ 移除未使用的信号和导入

### 4. 使用方法

**开发环境运行：**
```bash
pip install -r requirements.txt
python voiceink/main.py
```

**打包成 exe：**
```bash
pip install -r requirements.txt
python build.py
# 输出: dist/VoiceInk.exe
```

**使用步骤：**
1. 运行 VoiceInk.exe（或 python voiceink/main.py）
2. 首次运行会自动下载 Whisper base 模型（约150MB）
3. 右键托盘图标 → 打开设置 → 配置 LLM API（可选）
4. 按住 Alt+Space 说话，松开自动转写+润色+粘贴
5. 按 Esc 取消当前录音
