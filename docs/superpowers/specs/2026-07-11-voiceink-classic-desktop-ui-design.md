# VoiceInk UI · 经典桌面工具气质 — 设计文档

> 日期: 2026-07-11  
> 状态: 已确认  
> 关联: 覆盖 [UI Stitch 对齐设计](2026-07-10-voiceink-ui-stitch-alignment-design.md) 的**视觉结论**；托盘以 [托盘菜单风格](2026-07-10-voiceink-tray-menu-style-design.md) 为准并仅做 token 对齐  
> 背景: 在「经典耐看 / 高级感」审计后，选定方向 B（经典桌面工具）而非 Stitch Modern/Corporate 克制版

## 1. 目标与非目标

### 目标

将 VoiceInk 浅色 UI 与浮窗收成 **经典桌面工具** 气质：冷灰中性轴、主色克制、少盒子、层级靠留白与细分隔；设置 / 历史 / 浮窗 / 托盘微调一次对齐，避免气质分裂。

### 非目标

- 不改 ASR / 润色 / 粘贴主链路，不改 `App._connect_signals` 四条红线信号语义。
- 不像素级还原任何 HTML / Stitch 稿。
- 不引入暗色设置主题；不重做信息架构（页仍为：通用 / 模型 / 润色 / 关于 + 独立历史窗）。
- 不把托盘改回 Stitch 大圆角蓝悬停。
- 不覆盖功能文案与配置项语义；仅视觉与容器结构。

## 2. 已确认决策

| 决策点 | 结论 |
|--------|------|
| 气质 | 经典桌面工具（非 Stitch Corporate 克制版） |
| 主色 | 保留蓝，降饱和；仅主 CTA + 当前选中 |
| 范围 | 全表面一次收（设置 + 历史 + 浮窗 + token；托盘微调） |
| 结构 | 去盒化；交互卡保留；去掉通用页伪导航栏 |
| 浮窗状态色 | 中性字 + 仅「录音中」沉稳红强调 |
| 实现路径 | Token 先行，一次收口（方案 1） |

## 3. Design Token

以 `voiceink/ui/design_tokens.py` 为唯一色板 / 圆角 / 字体来源，驱动 `settings_styles.py` / `app_styles.py` / 各窗 QSS。

| 角色 | 拟议值 | 说明 |
|------|--------|------|
| App BG | `#F3F4F6` | 单一冷灰；去掉暖 pearl 与冷灰分裂 |
| Surface | `#FFFFFF` | 侧栏选中条、交互卡、必要面板 |
| Hairline / Border | `#E5E7EB` | 分区分隔，替代多数卡片边框 |
| Text | `#111827` / `#4B5563` / `#9CA3AF` | 主 / 次 / 弱三级 |
| Accent | `#2563EB`（hover `#1D4ED8`） | 比现 `#0050CB` / `#0066FF` 略收 |
| Accent soft | `rgba(37, 99, 235, 0.08)` | 仅选中底，不用大面积提示条 |
| Radius | 容器 8 · 控件 6 · 状态标签可用 pill | 托盘保持 ~4 |
| Font UI | `"Segoe UI Variable", "Microsoft YaHei UI", "Segoe UI", sans-serif` | Inter 移出优先 |
| Font mono | `"Cascadia Mono", "Consolas", monospace`（可保留 JetBrains 作可选回退） | 快捷键、路径 |
| 页标题 | ~20–22px / semibold | 不再 28px / 700 |
| 浮窗默认字 | 近白 / 次级浅灰 | 状态标题不换彩虹色 |
| 浮窗录音强调 | 沉稳红（如略降饱和的 `#EF4444` 族） | 仅录音中；点与波形跟此色 |
| 成功 / 危险 | 语义绿 / 红保留 | 仅成功提示、删除等；不用于浮窗状态轮换 |

### 主色限用规则（硬约束）

蓝只允许出现在：

1. 主 CTA 按钮  
2. 当前选中（左边条 **或** 细蓝边 + `accent soft` 底；避免蓝边 + 蓝底 + 蓝字三重强调）

导航文字、热键框、提示条、徽章、页头状态默认中性灰。

## 4. 架构与改动面

```
design_tokens.py          ← 冷灰轴 / 降饱和蓝 / 圆角 / 字体 / 浮窗状态色
settings_styles.py        ← WINDOW_CSS、按钮、输入 QSS
app_styles.py             ← 全局片段（若有）
settings_components.py    ← 去盒分区、侧栏、去掉 GeneralPageHeader、交互卡选中态
settings_window.py        ← 四页组装（不改配置读写语义）
history_window.py         ← 少套卡、操作主次、选中态
floating_window.py        ← 中性状态 + 仅录音强调色；交互语义不变
tray_icon.py              ← FONT / 灰阶对齐；保持原生菜单节奏
model_card.py / nav_icons ← 跟随新 token
tests/test_ui_styles.py 等 ← 契约与 token 断言按需更新
```

**数据流 / 信号：** 无变更。设置仍即时写配置；托盘信号不变；浮窗 `continuous_stop_requested` 等语义不变。

## 5. 界面规格

### 5.1 设置外壳

- 左：品牌行（图标 + VoiceInk）→ 简短状态两行（**无白卡片外壳**）→ 四项导航 → 底部短状态可保留或并入状态行。
- 导航选中：左边条 + 浅灰或极浅蓝底；**文字保持深色**（不再整项变蓝字）。
- 右：页标题 20–22px semibold + 可选一行副标题；内容滚动；底栏「更改会即时生效」为次要灰字 +「完成」主按钮。
- **删除** `GeneralPageHeader`（返回 / 居中蓝标题 / 齿轮）；通用页与其它页一样用侧栏 + `PageHero`。

### 5.2 通用

- **触发方式**：保留两张可点选卡；选中用左边条或细蓝边 + `accent soft`，禁止三重强调。
- 其后分区（快捷键、开机与提示音、历史保留、音频来源等）：**分区小标题 + hairline 分隔的设置行**，不再每节一张白卡。
- 热键框：中性底 + 中性边，等宽字体；焦点用 focus ring，不整框变蓝字。

### 5.3 模型

- 「当前引擎」：一行摘要（名 · 评级 · 大小），轻面板或无厚边框卡。
- 模型列表：列表项可保留轻边框（交互需要）；「当前」用中性小标签或左边条，不用高饱和蓝 pill。
- 存储路径：KV / 路径行 + 次要「更改」按钮，去盒。

### 5.4 润色

- 总开关行去盒；效果预览保留，原文/润色样例用浅底块，少一层外卡。
- API / 提示词：标题 + 分隔行。
- 页头「已开启 / 已关闭」用次要灰字，不用彩色 pill。

### 5.5 关于

- 品牌 + 版本：版本用中性小标签；去掉大块蓝用法提示条，改成一行次要说明。
- 运行信息：干净 KV 表，路径等宽；无营销感卡片堆叠。

### 5.6 历史窗口

- 左：标题 + 搜索 + 列表；选中浅底 + 左边条，文字深色。
- 右：顶栏操作分主次——导出为主或次主，复制* 为幽灵，删除为危险次要；详情为整栏内容面，少套白卡。
- `HistoryStore` API、搜索 debounce、导出 / 删除逻辑不变。

### 5.7 浮窗

- 深色底与交互保留。
- 标题与说明默认近白 / 次级灰。
- **仅「录音中」** 使用沉稳红（点、标题、波形）。
- 监听 / 识别 / 润色 / 成功 / 停止：中性色 + 文案；脉冲减弱或仅录音脉冲。
- 关闭钮：中性图标按钮；若可则细线图标替代粗 `×`；`continuous_stop_requested` / 关闭语义不变。

### 5.8 托盘菜单

- 保持 [托盘菜单风格设计](2026-07-10-voiceink-tray-menu-style-design.md)：小圆角、通栏分隔、浅灰悬停、左侧 ✓。
- 仅对齐新 FONT / 灰阶 token；勾选可用深灰或极克制 accent；**不要**蓝底悬停。

## 6. 错误处理与降级

- UI 重组不得引入新的阻塞路径；历史失败 / 空态 / 删除确认按现有行为。
- 字体缺失时回退 Segoe UI / 微软雅黑。
- Focus 优先 ring/outline，避免边框 1→2px 同步改 padding 导致微抖；QSS 做不到时以可读为底线。
- 浮窗中性化后，状态必须仍靠**文案**可辨（保留现有状态文案）。

## 7. 测试与验收

- 跑：`pytest tests/test_readme_features.py` 及 settings / history / floating / tray / ui_styles 相关用例；不得因纯视觉破坏契约。
- 人工验收：
  1. 设置四页：无伪导航栏；多数分区无白卡外壳；触发 / 模型交互卡仍在  
  2. 蓝仅出现在主按钮与选中态  
  3. 历史：选中左边条、操作主次分明、少套卡  
  4. 浮窗：仅录音为红强调，其余中性  
  5. 托盘：仍为原生小圆角灰悬停节奏  
- README：用户可见分区名 / 托盘文案无变则不必改；有文案级变化则按变更审查清单更新。
- 红线：不改 `ready` / `model_load_progress` / `segment_ready` / `esc_pressed` 语义。
