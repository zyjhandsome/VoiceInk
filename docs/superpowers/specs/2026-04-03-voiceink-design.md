# VoiceInk — 语音转文字桌面软件设计文档

> 日期：2026-04-03
> 状态：设计评审中

---

## 1. 概述

VoiceInk 是一款 Windows 桌面语音转文字软件。用户按住快捷键说话，松开后语音自动转为文字并粘贴到光标位置。内置本地 Whisper 语音识别引擎（离线运行），可选配置 LLM API 对转写文字进行润色，输出通顺的书面语。

### 1.1 设计原则

- **最少用户干预，直接给出最优结果**
- 开箱即用：不依赖网络即可完成基础语音转文字
- 渐进式体验：配置 LLM 后解锁润色功能
- 信息安全：语音数据全部本地处理，不上传第三方服务器

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| 按住说话 | 按住 Alt+Space 录音，松开自动处理 |
| 实时流式转写 | 边说边在悬浮窗显示文字 |
| LLM 自动润色 | 转写完成后自动润色为书面语（需配置 LLM） |
| 自动粘贴 | 结果自动粘贴到当前光标位置 |
| 降级运行 | 未配置 LLM 时直接输出原始转写 |
| 取消操作 | 录音中按 Esc 取消 |
| 声波动画 | 悬浮窗实时显示音量反馈 |
| 音效提示 | 开始/结束录音播放提示音 |
| 自定义快捷键 | 用户可在设置中修改快捷键 |
| 打包分发 | 使用 PyInstaller 打包为单个 .exe |

## 2. 技术栈

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 编程语言 | Python 3.11+ | 全栈统一 |

| 打包工具 | PyInstaller | 打包为 Windows .exe |

## 3. 系统架构

### 3.1 模块关系图

```
┌─────────────────────────────────────────────┐
│                  VoiceInk                    │
├──────────┬──────────┬──────────┬─────────────┤
│ 热键管理  │ 音频录制  │ 语音识别  │  LLM 润色   │
│ HotKey   │ Recorder │   STT    │  Polisher   │
│ Manager  │          │(Whisper) │             │
├──────────┴──────────┴──────────┴─────────────┤
│              文字输出（Auto-Paste）             │
├──────────────────────────────────────────────┤
│          UI 层（PyQt6）                        │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐ │
│  │ 系统托盘  │ │ 悬浮窗   │ │  设置主窗口   │ │
│  └──────────┘ └──────────┘ └──────────────┘ │
├──────────────────────────────────────────────┤
│          配置管理（Settings）                   │
└──────────────────────────────────────────────┘
```

### 3.2 数据流

1. 用户按住 Alt+Space → HotKeyManager 发出 `recording_start` 信号
2. 悬浮窗弹出，显示"录音中..." + 声波动画 → AudioRecorder 开始采集麦克风音频
3. 录音过程中，每积累约 3 秒音频 → SpeechRecognizer 分段转写 → 悬浮窗实时显示部分结果
4. 用户松开 Alt+Space → HotKeyManager 发出 `recording_stop` 信号 → AudioRecorder 停止录音
5. SpeechRecognizer 对完整音频做最终转写，修正流式阶段误差
6. 如已配置 LLM：悬浮窗显示"润色中..." → TextPolisher 调用 LLM API 润色
7. TextPaster 将最终文字粘贴到光标位置 → 悬浮窗显示"已输入"，1.5 秒后消失

取消流程：录音中按 Esc → 停止录音 → 丢弃音频 → 悬浮窗显示"已取消"，1 秒后消失

## 4. 模块详细设计

### 4.1 热键管理模块（HotKeyManager）

- **文件：** `hotkey_manager.py`
- **职责：** 全局快捷键监听，不受当前焦点窗口影响
- **技术：** pynput
- **信号：**
  - `recording_start` — Alt+Space 按下时发出
  - `recording_stop` — Alt+Space 松开时发出
  - `recording_cancel` — 录音中按 Esc 时发出
- **默认快捷键：** Alt+Space（注意：与 Windows 系统菜单快捷键冲突，软件运行时全局拦截此组合键；用户可在设置中自定义修改为其他组合键以避免冲突）
- **自定义快捷键：** 用户可在设置中修改，配置保存后实时生效，无需重启

### 4.2 音频录制模块（AudioRecorder）

- **文件：** `audio_recorder.py`
- **职责：** 麦克风音频采集，提供实时音量数据
- **技术：** sounddevice
- **参数：**
  - 采样率：16000 Hz（Whisper 标准输入）
  - 通道数：单声道
  - 格式：16-bit PCM
- **输出：**
  - 音频数据流 → 供 SpeechRecognizer 分段处理
  - 实时音量值 → 供悬浮窗声波动画显示

### 4.3 语音识别模块（SpeechRecognizer）

- **文件：** `speech_recognizer.py`
- **职责：** 将音频转为文字，支持实时流式输出
- **技术：** faster-whisper（CTranslate2 优化）
- **模型：**
  - 默认：base（约 150MB，平衡速度与精度）
  - 可选：small / medium / large（用户在设置中切换）
  - 首次使用时自动下载模型到 `~/.voiceink/models/`
- **流式处理策略：**
  - 录音过程中，每积累约 3 秒音频送入 Whisper 转写
  - 实时将部分结果通过信号传递给悬浮窗
  - 录音结束后，对完整音频做最终转写，修正流式阶段误差
- **语言：** 自动检测中英文

### 4.4 LLM 润色模块（TextPolisher）

- **文件：** `text_polisher.py`
- **职责：** 调用用户配置的 LLM API 润色文字
- **技术：** httpx（异步 HTTP）
- **API 格式：** 兼容 OpenAI Chat Completions API（支持 OpenAI、DeepSeek、通义千问、本地 Ollama 等）
- **用户配置项：**
  - API URL（如 `https://api.deepseek.com/v1`）
  - API Key
  - Model Name（如 `deepseek-chat`）
- **内置润色 Prompt：**
  ```
  请将以下语音转写文字润色为通顺的书面语，保持原意不变，修正口语化表达和语法错误，
  自动添加合适的标点符号。如果包含中英文混合内容，保持原有语言不变。
  只返回润色后的文字，不要任何解释。
  ```
- **降级策略：**
  - 未配置 LLM → 跳过润色，直接输出原始转写
  - LLM 请求超时（5 秒）→ 跳过润色，输出原始转写
  - LLM 请求失败 → 跳过润色，悬浮窗短暂提示错误

### 4.5 文字输出模块（TextPaster）

- **文件：** `text_paster.py`
- **职责：** 将最终文字粘贴到用户当前光标位置
- **技术：** pyperclip + pyautogui + win32gui（Windows API）
- **流程：**
  1. 通过 `win32gui.GetForegroundWindow()` 检查当前是否有活动窗口
  2. 若无活动窗口或活动窗口为 VoiceInk 自身：将文字写入剪贴板，悬浮窗提示"已复制到剪贴板"
  3. 若有活动窗口：备份剪贴板 → 写入结果 → 延迟 100ms → 模拟 Ctrl+V → 恢复剪贴板
- **依赖：** 新增 pywin32 用于 Windows 窗口检测

## 5. UI 设计

### 5.1 系统托盘

- **图标：** 麦克风图标（正常灰色，录音中红色）
- **右键菜单：**
  - 打开设置
  - 分隔线
  - 开机自启（勾选项）
  - 退出

> 注：历史记录功能不在首版范围内，后续版本按需迭代。

### 5.2 悬浮窗

- **外观：** 圆角半透明窗口（约 320×100px），屏幕底部居中，深色毛玻璃背景，白色文字
- **状态流转：**
  1. **录音中** — 显示"录音中..." + 实时声波动画 + 实时流式文字（边说边出字）
  2. **识别中** — 显示"识别中..." + 对完整音频做最终转写（修正流式阶段结果）
  3. **润色中** — 显示"润色中..." + 最终转写文字（仅配置 LLM 时出现此阶段）
  4. **已输入** — 成功提示，1.5 秒后自动消失
  5. **已取消** — 取消提示，1 秒后自动消失
  6. **已复制到剪贴板** — 无活动窗口时的提示，2 秒后消失
  7. **错误提示** — 如"识别失败"、"未检测到麦克风"、"润色失败，已输出原文"，2 秒后消失

### 5.3 设置主窗口

- **窗口：** 约 500×450px，不可调整大小，简洁现代风格
- **标签页：**

**【基础设置】**
- 快捷键：显示当前快捷键 + 修改按钮
- 开机自启动：勾选框
- 提示音：勾选框（开始/结束录音时播放提示音）

**【语音识别】**
- Whisper 模型选择：base（推荐）/ small / medium / large
- 模型状态：显示已下载/未下载，提供下载按钮

**【LLM 润色】**
- 启用 LLM 润色：勾选框
- API URL：文本输入框
- API Key：密码输入框 + 显示/隐藏切换
- Model Name：文本输入框
- 测试连接按钮

## 6. 配置管理

- **配置文件位置：** `~/.voiceink/config.json`
- **Whisper 模型缓存：** `~/.voiceink/models/`
- **配置结构：**

```json
{
  "hotkey": "alt+space",
  "auto_start": false,
  "sound_enabled": true,
  "whisper_model": "base",
  "llm": {
    "enabled": false,
    "api_url": "",
    "api_key": "",
    "model_name": ""
  }
}
```

## 7. 错误处理

| 场景 | 处理方式 |
|------|---------|
| 麦克风不可用 | 悬浮窗提示"未检测到麦克风" |
| Whisper 模型未下载 | 首次使用自动下载，显示下载进度 |
| Whisper 转写失败 | 悬浮窗提示"识别失败"，2 秒后消失 |
| LLM API 请求失败/超时 | 降级输出原始转写，悬浮窗短暂提示 |
| 无活动窗口可粘贴 | 通过 win32gui 检测前台窗口，若无目标窗口则保留文字在剪贴板，提示"已复制到剪贴板" |

## 9. 打包分发

- **工具：** PyInstaller
- **目标：** 单个 .exe 文件（`--onefile` 模式）
- **包含内容：** Python 运行时 + 所有依赖 + 资源文件（图标、音效）
- **Whisper 模型：** 默认不打包进 exe（体积考虑），首次运行时自动下载到用户目录
- **预计体积：** 约 200-300MB（不含模型）
