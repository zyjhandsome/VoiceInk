# 持续转写：默认 Ctrl+Space + 悬浮窗延后

## 背景

持续模式下短按快捷键会弹出「待开始 / 按住 … 开始持续监听」，体感像一按就出窗。默认快捷键改为 Ctrl+Space，并推迟悬浮窗出现时机。

## 决策

1. **默认快捷键** `ctrl+space`（仅影响新装/无该字段的配置；已有 `config.json` 不强制迁移）。
2. **持续模式短按**：不调用 `show_continuous_idle`；托盘提示保留（限频）。
3. **模型就绪 / 设置变更 / 热键更新**：持续模式不再主动弹「待开始」浮窗；托盘仍可提示就绪。
4. **浮窗出现时机**：真正开始监听、识别中、出字/错误、用户停止等过程态。
5. **按住说话模式**：短按仍可弹「录音过短」类提示（阈值 0.18s）。
6. **文档**：README 默认键改为 Ctrl+Space；说明若与输入法冲突可改 Alt+Space。

## 非目标

- 不改按住达标后的「监听中」浮窗。
- 不强制改写用户已保存的快捷键。
