# ADR-0007 · 历史总开关的运行时行为

> 日期:2026-07-10 · 状态:已接受 · 关联:spec §3.5/§3.6、决策补丁 D1
> 来源:grill-with-docs 拷问 Question 7(开关运行中切换)

## 决策

1. **判定时机**:`history.enabled` 在**每次入队那一刻**读取,不启动时缓存。用户一关,
   **下一段起立即停止写入**,即使正处于持续会话中途。
2. **关闭时已有数据**:**只停未来写入,已有历史全部保留**。停止记录与删除历史是两个独立动作,
   避免「总开关误触即毁数据」。
3. **独立「清空全部历史」动作**:历史窗口提供显式「清空全部」按钮(带二次确认),与总开关解耦。
   总开关只管「记不记」,清空是另一个明确入口。

## 影响

- 写入路径每次 `enqueue` 前检查 `config.get("history.enabled")`;`false` 直接跳过(连 `session_id` 也不必建)。
- 「清空全部」是写操作,经后台写线程(ADR-0006)。
- 测试:开关关闭时任何分支不写;运行中关闭后下一段不写、已有行不动;清空全部经写线程且需确认。
