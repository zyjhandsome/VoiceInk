# ADR-0005 · 自动清理按「会话」计量与执行

> 日期:2026-07-10 · 状态:已接受 · 关联:spec §3.5/§3.6、决策补丁开放问题 #4
> 来源:grill-with-docs 拷问 Question 5(自动清理语义)

## 背景

方案 C 下一行 = 一个 segment,一场会话含多行。spec 的「90 天 / 5000 条(先到者)」
若不定清「条」与「年龄」的单位,行为会反直觉,且可能半删一场会话破坏拼接/导出。

## 决策

1. **`max_entries` 按会话计数**:`COUNT(DISTINCT session_id)`;超限时**整场删除最旧会话**,
   直到回到上限内。默认 `5000` = 5000 场会话。
2. **`retention_days` 按会话 `MIN(created_at)` 计龄**:一场会话所有行同龄,
   **要么整场保留要么整场删除,绝不半删**。默认 90 天。
3. **触发时机**:启动时跑一次 + 每场会话结束后跑一次;`COUNT`/`DELETE` 全走后台写线程,
   不碰主/粘贴路径。**不**每段触发。
4. **活跃会话豁免**:清理永不删当前活跃 `session_id`(否则正在写入 + 迟到分段回插会状态错乱)。
5. 「先到者」:天数与条数上限任一触发即清理。

默认值维持 **90 天 / 5000 场会话**(用户确认无需调整)。

## 影响

- `HistoryStore` 需提供以会话为单位的清理:先选出待删 `session_id` 集合(按龄/超量,排除活跃),
  再 `DELETE ... WHERE session_id IN (...)`。
- 清理是写操作,必须与 INSERT 一样走后台写线程(见 ADR-0003 / ADR-0006 单写者原则)。
- 测试:按会话计数删除、按会话计龄删除、不半删会话、活跃会话不被删、先到者语义。
