## Context

`4fc51d8` 已将浅色 UI 收成「经典桌面工具」：冷灰轴、蓝限用、去盒化。用户反馈设置界面仍偏 Low。已确认方向：**经典抬质感 + 设置壳产品感**（非全表面换气质）。

工作区已有未提交 UI 微调（分组白底、导航去左边条、开关绿轨等）及大量无关文档删除。实现须只吸收对齐本规格的 UI 改动，忽略无关删除。

## Goals / Non-Goals

**Goals:**

- 设置壳具备可感知的产品感表面层次（侧栏 / 内容 / 分组 / 底栏）。
- 导航与触发卡选中态清晰，且遵守蓝限用（禁止三重强调）。
- 焦点环与主 CTA 仍可读；配置语义不变。
- 更新相关 UI 契约测试。

**Non-Goals:**

- 不改 ASR / 润色 / 粘贴 / 历史存储 / 信号红线。
- 不重做信息架构或设置文案。
- 不引入暗色设置主题或新依赖。
- 本变更不重设计浮窗 / 历史窗 / 托盘视觉语言。

## Decisions

1. **Token 先行，设置壳优先消费**  
   - 在 `design_tokens.py` 增补/调整设置壳所需表面、选中洗色、间距；共享 token 变更必须跑 settings + ui_styles，并目视确认 float/history/tray 无主题冲突。  
   - 备选：仅改局部 QSS —— 拒绝，易与经典轴再次分裂。

2. **产品感 = 受控白底分组 + 清晰选中，而非大阴影堆叠**  
   - `settings_group` 使用 `SURFACE` + 圆角；画布保持 `BG`；分区标题保持次要灰字。  
   - Qt/QSS 阴影能力弱：优先边框/对比/留白节奏；若用阴影仅允许极轻、单层。  
   - 备选：恢复 Stitch 大面积蓝与厚卡 —— 拒绝，与已确认「经典+产品感」冲突。

3. **导航选中：单一蓝强调 + 中性洗色**  
   - 允许左边条 **或** 细蓝指示，配合中性/极浅蓝底；选中文字保持深色。  
   - WIP 去掉左边条后若选中不够明显，应恢复单一指示而不是加蓝字。

4. **触发卡：保留双卡交互，收敛选中样式**  
   - 继续 `TriggerModePicker` / `VerticalChoiceCard`；选中 = 左边条或细边 + soft wash。

5. **未提交冲突策略**  
   - 对齐本设计的现有 diff 可保留并整理；无关 `docs/` / `openwiki/` 删除不纳入本变更提交范围。

## Risks / Trade-offs

- [共享 token 波及浮窗/历史] → 变更后跑相关测试并人工抽查；禁止为本变更顺手重做它们。  
- [「产品感」主观] → 以规格场景（层次、选中、蓝限用、焦点）作验收，不做开放式美化。  
- [WIP 与 HEAD 混杂] → 实现前对照 `git status`，只编辑允许路径。  
- [独立评审] → Medium 风险；实现完成后需独立 Agent/人工 review 才可宣称完成。

## Migration Plan

- 无数据迁移；纯视觉/QSS/组件。  
- 回滚：还原本变更涉及的 UI 文件与测试断言即可。
