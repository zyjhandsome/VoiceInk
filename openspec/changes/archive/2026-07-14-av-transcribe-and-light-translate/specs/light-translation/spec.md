## Purpose

定义文件转写任务上的事后轻量翻译行为：与润色互斥、复用现有 OpenAI 兼容 API 路径变体、失败回退原文。明确不适用于实时听写路径，也不构成同声传译。

## ADDED Requirements

### Requirement: 文件转写任务可启用轻翻译

系统 SHALL 允许用户为**文件转写任务**启用轻翻译（目标语言可配置）。实时听写路径 MUST NOT 因本能力而自动进入翻译。

#### Scenario: 文件任务翻译开启

- **WHEN** 用户已配置可用的 OpenAI 兼容 API，启用翻译并选定目标语言，且文件转写得到非空原文
- **THEN** 系统在文件任务完成后产出目标语言译文供用户查看/使用

#### Scenario: 实时听写不翻译

- **WHEN** 用户通过热键完成实时听写（即使设置中存在翻译相关选项）
- **THEN** 系统 MUST NOT 将该次听写结果按本能力自动翻译（仍可按既有逻辑润色或输出原文）

### Requirement: 翻译与润色互斥

对同一段输出，系统 SHALL 在「润色」与「翻译」之间互斥；启用翻译时 MUST NOT 再对该段应用润色语义。译文存储 MAY 复用既有 `polished_text` 槽位，MUST NOT 要求新增历史 schema 列作为本变更前提。

#### Scenario: 选择翻译时不润色

- **WHEN** 文件任务启用翻译模式
- **THEN** 系统对该段输出执行翻译而不执行润色 prompt

#### Scenario: 选择润色时不翻译

- **WHEN** 文件任务未启用翻译且润色开启
- **THEN** 系统按既有润色行为处理，MUST NOT 改变原文语言

### Requirement: 翻译失败回退原文

翻译失败时，系统 SHALL 回退为 ASR 原文（或已得到的未翻译文本），并向用户给出可感知提示；MUST NOT 丢弃原文。

#### Scenario: API 失败回退

- **WHEN** 翻译 API 请求失败或返回空/无效结果
- **THEN** 系统使用原文作为该次输出，并提示翻译未成功（降级）

#### Scenario: 未配置 API 时不可伪造成功

- **WHEN** 用户启用翻译但 API 未正确配置
- **THEN** 系统 MUST 在开始翻译前或失败时提示配置问题，且 MUST NOT 将未翻译文本标为已翻译成功

### Requirement: 翻译结果可进入历史

当文件转写启用翻译且成功时，历史记录 SHALL 保存原文与译文（译文可位于既有 `polished_text` 字段）；用户 MUST 能区分或查看二者（至少能看到最终展示文本，且原文可查）。

#### Scenario: 成功翻译写入历史

- **WHEN** 文件转写与翻译均成功且历史已启用
- **THEN** 历史记录包含 ASR 原文，并在润色/译文槽位保存译文

#### Scenario: 翻译降级写入

- **WHEN** 翻译失败并回退原文
- **THEN** 历史仍保存原文；MUST NOT 将原文误标为成功译文（若有状态/提示，须反映降级）

### Requirement: 非同声传译边界

本能力 SHALL 仅提供事后（转写完成后）轻翻译，MUST NOT 宣称或实现真·流式同声传译、会议级低延迟双语叠字。

#### Scenario: 产品语义边界

- **WHEN** 用户使用翻译相关设置或结果面
- **THEN** 系统行为与文案 MUST 体现「转写后翻译」，MUST NOT 将本能力描述为同声传译
